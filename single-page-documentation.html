# Single Page Documentation ## With the styles of BassCSS This is the skeleton for a single page documentation. It comes with a nice basscss layer wich you can use in the body in the html language. ## MarkDown + HTML Additionaly to HTML, you can write in markdown, thanks to the [markdown-it] javascript library. ## Responsive This documentation can easily be viewed in any mobile device. When the screen is big enough, it will nicely split and a left navbar will appear. ### Auto-generated navbar A navbar on the left of the screen will be generated automatically with the contents of the document. The contents include any `h1` to `h6` tag you place in the body. This is achieved thanks to [anchorific], whose [beautifull example][anchorific example] shows the full potential of auto-generating nested navigations and anchor links on title tags. # With Sintax Highlight Did I mention sintax highlight yet? It's powered by [Highlight.js]. ``` var Example = new Documentation( { 'documentation': true, 'basscss': true, 'anchorific': true } ); ``` It uses monokai theme by default, but you could easily change that: (line 60) ``` ``` If you choose a light theme, you will also need to delete the css rule at line 20: ``` pre{ background-color: #212021; color: #e1e1e1; } ``` By default, the highlight sintax is configured to _javascript_, yet, I made it super easy for you to change that. Simply change line 65 to whatever language you prefer. ``` var defaultHighlightLanguage = 'javascript'; // Leave NULL ('') to let Highlight.js auto-detect the languaje ``` [markdown-it]: https://github.com/markdown-it/markdown-it [anchorific]: https://github.com/renettarenula/anchorific.js/ [anchorific example]: http://renaysha.me/anchorific-js/ [Highlight.js]: https://highlightjs.org/
